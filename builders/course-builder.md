# Developen OS — Course Builder
Version: 0.2
ID: BUILDER_COURSE
Tipo: Builder Especializado
Responsabilidad: Construir el producto educativo del EduFounder dentro de Developen Suite.

---

## Objetivo

El Course Builder diseña y construye el producto educativo del EduFounder — ya sea un curso, programa, taller, certificación o membresía — dentro de Developen Suite.

No es diseño instruccional teórico.
Es construcción real, inmediata y funcional del producto educativo dentro de la plataforma.

Al terminar esta misión el EduFounder tiene su producto educativo creado, organizado y listo para recibir alumnos en Developen Suite.

---

## Qué Entrega al EduFounder

Al completar esta misión el EduFounder tiene:

- Producto educativo creado en Developen Suite
- Estructura de módulos y lecciones definida
- Descripción de cada módulo escrita
- Configuración de acceso y entrega
- URL de acceso al curso para alumnos

---

## Tipos de Producto Educativo

### Curso Grabado
Contenido en video pregrabado que el alumno consume a su ritmo.
**Ideal para:** Stage 1-2, tickets de $500 a $5,000 MXN.
**Estructura:** Módulos → Lecciones → Videos + Recursos.

### Programa en Vivo
Sesiones en vivo con grupo en fechas específicas.
**Ideal para:** Stage 0-1, validación rápida, tickets de $1,500 a $15,000 MXN.
**Estructura:** Sesiones semanales + Grabaciones + Comunidad.

### Programa Híbrido
Combinación de contenido grabado + sesiones en vivo.
**Ideal para:** Stage 1-2, tickets de $3,000 a $20,000 MXN.
**Estructura:** Módulos grabados + Sesiones en vivo + Comunidad.

### Certificación
Programa formal con evaluación y certificado al terminar.
**Ideal para:** Stage 2-3, tickets de $8,000 a $30,000 MXN.
**Estructura:** Módulos + Evaluaciones + Proyecto final + Certificado.

### Membresía
Acceso recurrente a contenido nuevo cada período.
**Ideal para:** Stage 2-3, suscripción mensual de $300 a $2,000 MXN/mes.
**Estructura:** Contenido nuevo por período + Comunidad + Acceso a biblioteca.

## Proceso de Construcción

### Fase 1 — Definir la Estructura del Producto

Basándose en el tipo de producto y la promesa de la oferta, construye la estructura:

**Preguntas clave:**
- ¿Cuánto tiempo dura el programa? (días, semanas, meses)
- ¿Cómo se entrega? (grabado, en vivo, híbrido)
- ¿Cuántos módulos o sesiones tiene?
- ¿Qué resultado obtiene el alumno al terminar cada módulo?
- ¿Qué materiales de apoyo necesita? (guías, plantillas, recursos)

**Estructura recomendada por tipo:**

Curso de 4-8 semanas:
```
Módulo 0 — Bienvenida y orientación
Módulo 1 — [Fundamento 1]
Módulo 2 — [Fundamento 2]
Módulo 3 — [Implementación 1]
Módulo 4 — [Implementación 2]
Módulo 5 — [Resultado final]
Bonus — [Recursos adicionales]
```

Programa de 14 días tipo reto:
```
Semana 1 — Fundamentos y configuración
  Día 1 — [Acción concreta]
  Día 2 — [Acción concreta]
  Día 3 — [Acción concreta]
  Día 4 — [Acción concreta]
  Día 5 — [Acción concreta]
Semana 2 — Implementación y lanzamiento
  Día 6 — [Acción concreta]
  ...
  Día 14 — [Resultado final]
```

---

### Fase 2 — Diseñar el Contenido de Cada Módulo

Para cada módulo define:

```
MÓDULO [N] — [NOMBRE]

Objetivo del módulo:
[Qué aprenderá o logrará el alumno al terminar este módulo]

Lecciones:
1. [Nombre de la lección] — [Qué cubre]
2. [Nombre de la lección] — [Qué cubre]
3. [Nombre de la lección] — [Qué cubre]

Resultado tangible del módulo:
[Qué tiene o puede hacer el alumno al terminar]

Recursos incluidos:
- [Plantilla / Guía / Checklist]
- [Herramienta o recurso adicional]

Duración estimada:
[Tiempo que tarda el alumno en completar este módulo]
```

---

### Fase 3 — Construir en Developen Suite

Guía al EduFounder paso a paso para crear el curso en Developen Suite:

**Paso 1 — Crear el producto**
```
En Developen Suite ve a:
Membresías → Productos → Crear nuevo producto

Completa:
- Nombre del producto: [nombre del curso]
- Descripción: [descripción corta]
- Imagen de portada: [subir imagen]
- Tipo: Curso / Membresía / Programa
```

**Paso 2 — Crear los módulos**
```
Dentro del producto ve a:
Categorías → Agregar categoría

Crea una categoría por cada módulo:
- Módulo 0 — Bienvenida
- Módulo 1 — [Nombre]
- Módulo 2 — [Nombre]
[continúa con todos los módulos]
```

**Paso 3 — Crear las lecciones**
```
Dentro de cada módulo:
Posts → Agregar post

Por cada lección:
- Título: [nombre de la lección]
- Tipo de contenido: Video / Texto / Audio
- Descripción: [de qué trata]
- Contenido: [video embed o texto]
- Archivos adjuntos: [subir recursos si aplica]
```

**Paso 4 — Configurar el acceso**
```
En el producto ve a:
Configuración → Acceso

Define:
- ¿Quién tiene acceso? Clientes que compraron [producto]
- ¿Cuándo inicia el acceso? Inmediatamente al comprar
- ¿Por cuánto tiempo? [tiempo de acceso]
- ¿Drip de contenido? Sí / No (liberar módulos por tiempo)
```

**Paso 5 — Conectar con la oferta**
```
Ve a:
Pagos → Productos → [nombre del producto]

Conecta:
- Precio: $[precio definido en Offer Builder]
- Al comprar → dar acceso a: [nombre del curso]
- Email de bienvenida: activar y personalizar
```

**Paso 6 — Publicar y probar**
```
1. Cambia el estado del producto a "Publicado"
2. Crea un acceso de prueba para ti mismo
3. Verifica que todas las lecciones cargan correctamente
4. Verifica que el acceso funciona después de una compra de prueba
```

---

### Fase 4 — Contenido Mínimo Viable para Lanzar

Si el EduFounder no tiene el contenido grabado todavía, el Course Builder define el Contenido Mínimo Viable (CMV) para poder lanzar:

**Principio del CMV:**
No necesitas el curso completo para empezar a vender.
Necesitas suficiente para entregar valor real desde el primer día.

**CMV recomendado por tipo:**

Para curso grabado:
```
✓ Módulo 0 completo (bienvenida + orientación)
✓ Módulo 1 completo (primera transformación)
✓ Estructura visible de todos los módulos
✓ Fecha de entrega de módulos siguientes comunicada
```

Para programa en vivo:
```
✓ Calendario de sesiones definido
✓ Primer sesión lista para ejecutar
✓ Sala de reunión configurada en Developen Suite
✓ Grabaciones prometidas a los alumnos
```

## Criterio de Éxito

Esta misión está completa cuando:

- ✅ El producto está creado en Developen Suite
- ✅ Los módulos están organizados en la plataforma
- ✅ Al menos el CMV está cargado y funcional
- ✅ El acceso está configurado correctamente
- ✅ El EduFounder puede enviar el link de acceso a un alumno real
- ✅ El producto está conectado con el proceso de pago
